Agentic RAG Transformer - ML, Sci-Fi, and Cosmos Assistant

Introduction

rag is an intelligent retrieval-augmented generation system that helps you get answers from your knowledge base using advanced ML techniques.

Installation

Recommended setup (creates venv/ and installs dependencies):

bash SETUP.sh

Manual (editable install):

python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install -e .

Quick Start

Run the TUI (Terminal User Interface):

rag-tui

Or run the CLI:

rag --query "What is machine learning?"

Type help inside the TUI to see available commands.

Configuration

The app auto-loads a repo-root .env file (gitignored). Start with:

cp .env.example .env

Common variables:

RAG_LLM_BACKEND=local|openai|cerebras|ollama
RAG_MEMORY_MODE=off|session|persist
RAG_ENABLE_WEB=0|1

LLM Backends

Select a backend via RAG_LLM_BACKEND in .env or the shell.

OpenAI (GPT‑5.3)

RAG_LLM_BACKEND=openai
OPENAI_API_KEY=...
OPENAI_MODEL=gpt-5.3-chat-latest

Cerebras

RAG_LLM_BACKEND=cerebras
CEREBRAS_API_KEY=...
CEREBRAS_MODEL=llama3.1-8b
# or: qwen-3-235b-a22b-instruct-2507

Ollama (local)

ollama pull llama3
RAG_LLM_BACKEND=ollama
OLLAMA_MODEL=llama3

TUI Commands

Helpful commands inside rag-tui:

models           # show current backend/model and available models (if supported)
model            # pick a model from a list (Cerebras/Ollama)
model: <name>     # switch model
backends         # list available backends
backend          # pick backend from a list
backend: <name>   # switch backend
refresh          # reload engine/session
cache:clear      # delete project .cache (confirmation required)
memory:clear     # clear conversation memory
hf:clear         # delete Hugging Face cache (~/.cache/huggingface) (confirmation required)

Release Workflow

A single push to main triggers the release job, running semantic-release with the GitHub Actions bot.

Process

  • Checkout the repository with full history.
  • Spin up Python 3.14, upgrade pip, install requirements, and prepare wheel.
  • Configure git as the GitHub Actions bot to create tags, changelog entries, and the GitHub release.
  • Run semantic-release, then run python scripts/enhance_changelog.py.
  • Run python scripts/update_webpage_version.py to sync version in HTML.

Troubleshooting

Keep your local run minimal: always activate a virtual environment, install the package with pip install -e ., and run rag-tui from the same shell to ensure the entry points resolve correctly.

Common error: ModuleNotFoundError

If you see ModuleNotFoundError: No module named 'rag', run:

pip install -e .
